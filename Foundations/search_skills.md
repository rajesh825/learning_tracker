# Search Engines for Security
## Google Search Operators (For OSINT & Research)


### `site:`

Limits search results to a specific domain.

Example:  
`site:github.com security tools`

### `-` (Minus Sign)

Excludes words or phrases from search results.

Example:  
`pyramids -tourism` or `-tourism pyramids`

### `filetype:`

Searches for specific file types.

Example:  
`filetype:pdf cyber warfare report`


## Specialized Security Search Engines

### Shodan

Searches for Internet-connected devices (servers, routers, IoT, ICS).

- **Example query:** `lighttpd`
    
- **Top country for lighttpd servers:** United States

### Have I Been Pwned (HIBP)

Checks if an email address appears in known data breaches.

## Vulnerabilities and Exploits

### Common Vulnerabilities and Exposures (CVE)

Standardized identifiers for security vulnerabilities.

- **Format:** `CVE-2024-29988`
    
- **Maintained by:** MITRE Corporation
    

### National Vulnerability Database (NVD)

U.S. government repository of vulnerability data, based on CVE.

### Exploit Database

Public archive of verified exploit codes for security testing.

- **Use case:** Finding working exploit code for authorized penetration testing.
    
- **Important:** Only use exploits with proper authorization and legal agreement.
    

### Censys

Focuses on Internet hosts, websites, certificates, and Internet assets.

- **Comparison:** Shodan → devices; Censys → hosts & certificates.
    

### VirusTotal

Scans files, URLs, and hashes using multiple antivirus engines.

- **Example hash lookup:**  
    `2de70ca737c1f4602517c555ddd54165432cf231ffc0e21fb2e23b9dd14e7fb4`
    
- **Result:** `Android.Riskware.Agent.LHH`
